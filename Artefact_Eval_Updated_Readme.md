# Contents of the Artifact

In our submission, we have two separate components that we are applying for badge for:
1. **Model Cleansing Pipeline for Conceptual Models**: This is a Python library that provides functionalities for loading, filtering, extracting, and deduplicating datasets of conceptual models. It is designed to facilitate the cleansing of conceptual models in various domains, such as UML and ArchiMate.
2. **Reproducibility Package**: This package contains the code and data necessary to reproduce the results presented in our paper. It includes a comparative evaluation with an existing work, providing a comprehensive setup for researchers to validate our findings.


3. The reproducibility package can be downloaded from the Zenodo link provided in the STATUS file.
Publicly archived on Zenodo ([10.5281/zenodo.16285770] (https://doi.org/10.5281/zenodo.16285770))
A reproducibility package that contains the code and data to reproduce the results of the paper. 
The reproducibility package is structured as follows:
   - `reproducibility_package/` - This directory contains all the code and data needed to reproduce the results of the paper.
   - Instructions for running both reproducibility studies are provided in the README files within the `reproducibility_package/` directory.
4. `model-cleansing` folder which contains the MCP4CM (`mcp4cm`) python library can be downloaded from the Zenodo link provided in the STATUS file.
Publicly archived on Zenodo DOI: [10.5281/zenodo.16365384](https://doi.org/10.5281/zenodo.16365384)
The model-cleansing folder contains the Python library `mcp4cm` which provides functionalities for model cleansing. The library is structured as follows:
   - `mcp4cm/` - This directory contains the core library code.
   - `test_mcp4cm.ipynb/` - This directory contains unit tests for the library.
   - `dataset_generation.ipynb/` - This directory contains example notebooks demonstrating how to use the library for dataset generation.
   - `README.md` - This file provides an overview of the library, installation instructions, data downloading and usage examples.

We are applying for "Artifact Reusable" badges for our artefact -
    - Part 1: Model Cleansing Pipeline for Conceptual Models - this part is implemented as a Python library and will be publicaly available on Zenodo as well as a python package on PyPI.
    - Part 2: A reproducibility package for comparative evaluation with an existing work involved in our paper. The reproducibility package is available as a separate folder in this directory.
